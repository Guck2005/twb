Exercice 1 — QCM Administration Oracle

Tu peux aussi t'entraîner sur ces questions en mode interactif dans l'onglet QCM.

1
Définitions de poste d'un DBA Oracle ?

Réponse : D) Tout ce qui précède — Le DBA installe et configure le logiciel, met en œuvre la stratégie de sauvegarde/restauration et gère les utilisateurs et leurs privilèges.

2
Utilité d'un Tablespace ?

Réponse : D) Toutes les réponses — Un tablespace organise tables et index en groupes logiques, peut être mis en lecture seule (les données ne changent plus) et permet le transport de données (transportable tablespaces).

3
Composants de l'architecture Oracle ?

Réponse : D) Toutes les réponses sont bonnes — L'architecture Oracle comprend les processus d'arrière-plan, les datafiles (structure physique) et les tablespaces (structure logique).

4
Collection d'informations stockées à un moment donné ?

Réponse : A) Instance de la base de données — L'ensemble des informations stockées dans la base à un instant précis est appelé instance (état) de la base de données. Le schéma en est la description structurelle.

5
Tables physiquement stockées ensemble partageant une colonne commune ?

Réponse : D) Cluster — Un cluster regroupe physiquement plusieurs tables qui partagent une ou plusieurs colonnes communes (clé de cluster).

6
Regrouper des données de manière logique ?

Réponse : B) Tablespace — Le tablespace est l'unité de stockage logique d'Oracle (par opposition aux datafiles, physiques).

7
Quelle ligne produira une erreur ?

Réponse : D) Aucune de ces réponses — Les trois requêtes SELECT sont syntaxiquement correctes.

8
Qu'est-ce qu'un SGBD ?

Réponse : A) Collection de nombreux programmes pour accéder aux données — Un SGBD est un ensemble de logiciels permettant de créer, gérer et interroger des bases de données.

9
Avantages d'une vue ?

Réponse : D) Toutes les réponses sont vraies — Une vue offre des colonnes dérivées, renforce la sécurité des données et masque la complexité des requêtes.

10
Type de données de caractère inexistant ?

Réponse : D) String — String n'existe pas en Oracle. Les types caractères valides sont CHAR, VARCHAR/VARCHAR2 et NVARCHAR2.

11
Utilisation du « Library cache » ?

Réponse : B) Utilisé pour stocker le shared SQL — Le library cache (dans la Shared Pool de la SGA) conserve les requêtes SQL analysées et les plans d'exécution partagés.

12
Quels processus composent une instance Oracle ?

Réponse : D) Toutes les réponses sont vraies — Une instance Oracle = structures mémoire (SGA) + processus d'arrière-plan qui gèrent les données.

Exercice 2 — PL/SQL sur la base EventRequest

Schéma : Facility(facNo, facName) — Customer(custNo, custName, custContact, custPhone, custAddress, custCity, custState, custZip) — EventRequest(eventNo, facNo#, custNo#, dateHeld, dateReq, dateAuth, status, estCost, estAudience)

1
Procédure custProfile

Affiche toutes les caractéristiques du client dont le numéro est passé en paramètre.

CREATE OR REPLACE PROCEDURE custProfile (
    p_custNo IN Customer.custNo%TYPE
)
IS
    v_cust Customer%ROWTYPE;                          -- (1)
BEGIN
    SELECT *
    INTO v_cust
    FROM Customer
    WHERE custNo = p_custNo;                          -- (2)

    DBMS_OUTPUT.PUT_LINE('Numero   : ' || v_cust.custNo);
    DBMS_OUTPUT.PUT_LINE('Nom      : ' || v_cust.custName);
    DBMS_OUTPUT.PUT_LINE('Contact  : ' || v_cust.custContact);
    DBMS_OUTPUT.PUT_LINE('Tel      : ' || v_cust.custPhone);
    DBMS_OUTPUT.PUT_LINE('Adresse  : ' || v_cust.custAddress);
    DBMS_OUTPUT.PUT_LINE('Ville    : ' || v_cust.custCity);
    DBMS_OUTPUT.PUT_LINE('Etat     : ' || v_cust.custState);
    DBMS_OUTPUT.PUT_LINE('Zip      : ' || v_cust.custZip);
                                                      -- (3)
END;
/
2
Traitement d'exception (client inexistant)

Il suffit d'insérer une section EXCEPTION à l'endroit (3), juste avant le END;. Le SELECT ... INTO en (2) lève automatiquement NO_DATA_FOUND si le numéro n'existe pas :

-- Code a inserer en (3) :
EXCEPTION
    WHEN NO_DATA_FOUND THEN
        DBMS_OUTPUT.PUT_LINE(
            'Le client ' || p_custNo || ' n''existe pas.'
        );
        -- ou : RAISE_APPLICATION_ERROR(-20001, 'Client inexistant');
3
Bloc anonyme — affichage trié par coût estimé décroissant 
BEGIN
    FOR r IN (
        SELECT e.eventNo, f.facName, c.custName, e.estCost, e.status
        FROM EventRequest e
        JOIN Facility f ON f.facNo  = e.facNo
        JOIN Customer c ON c.custNo = e.custNo
        ORDER BY e.estCost DESC
    ) LOOP
        DBMS_OUTPUT.PUT_LINE(
            r.eventNo  || ' - ' || r.facName || ' - ' ||
            r.custName || ' - ' || r.estCost || ' - ' || r.status
        );
    END LOOP;
END;
/
4
Déclencheur eventCheck

Vérifie à chaque insertion ou modification que dateAuth est renseignée, puis que dateReq ≤ dateAuth ≤ dateHeld.

CREATE OR REPLACE TRIGGER eventCheck
BEFORE INSERT OR UPDATE
ON EventRequest
FOR EACH ROW
BEGIN
    IF :NEW.dateAuth IS NULL THEN
        RAISE_APPLICATION_ERROR(
            -20001, 'Erreur d''autorisation'
        );
    END IF;

    IF :NEW.dateAuth < :NEW.dateReq
       OR :NEW.dateHeld < :NEW.dateAuth THEN
        RAISE_APPLICATION_ERROR(
            -20002, 'Erreur de date'
        );
    END IF;
END;
/
5
Bloc anonyme — création d'une vue par établissement

On utilise du SQL dynamique (EXECUTE IMMEDIATE). Les noms d'établissements pouvant contenir des espaces, on les remplace par des underscores (ou on entoure le nom de guillemets doubles).

BEGIN
    FOR f IN (SELECT facNo, facName FROM Facility) LOOP
        EXECUTE IMMEDIATE
            'CREATE OR REPLACE VIEW facility' ||
            REPLACE(f.facName, ' ', '_') ||
            ' AS SELECT eventNo, status, estCost, estAudience' ||
            ' FROM EventRequest WHERE facNo = ' || f.facNo;
    END LOOP;
END;
/

Exercice 3 — Trigger de contrôle de capacité disque

Schéma : DISQUE(CodeDisque, nom, capacite, vitesse, fabricant) — PARTITION(NumPartition, nomPartition, taille, #CodeDisque)

1
Principe

Avant chaque insertion dans PARTITION, on calcule la somme des tailles des partitions déjà présentes sur le disque concerné, on y ajoute la taille de la nouvelle partition, et on compare à la capacité du disque. Si la capacité est dépassée, on lève une erreur : l'insertion est annulée.

2
Code du trigger 
CREATE OR REPLACE TRIGGER verif_taille_partition
BEFORE INSERT ON PARTITION
FOR EACH ROW
DECLARE
    v_capacite     DISQUE.capacite%TYPE;
    v_taille_totale NUMBER;
BEGIN
    -- Capacite du disque concerne
    SELECT capacite
    INTO v_capacite
    FROM DISQUE
    WHERE CodeDisque = :NEW.CodeDisque;

    -- Taille deja occupee par les partitions existantes
    SELECT NVL(SUM(taille), 0)
    INTO v_taille_totale
    FROM PARTITION
    WHERE CodeDisque = :NEW.CodeDisque;

    -- Verification (y compris la nouvelle partition)
    IF v_taille_totale + :NEW.taille > v_capacite THEN
        RAISE_APPLICATION_ERROR(
            -20010,
            'Insertion impossible : la taille totale des partitions (' ||
            (v_taille_totale + :NEW.taille) ||
            ') depasse la capacite du disque (' || v_capacite || ').'
        );
    END IF;
END;
/

Remarque : RAISE_APPLICATION_ERROR interrompt l'instruction INSERT en cours, donc l'enregistrement n'est pas effectué et le message d'anomalie est affiché.

Exercice 4 — Transactions (COMMIT / ROLLBACK)

Table ETIQUETTES(Et1, Et2) initialement vide.

1
Trace pas à pas

Étape 1 — 5 insertions : (A,A) (A,B) (A,C) (B,A) (B,B), puis ROLLBACK → table vide (rien n'était validé).

Étape 2 — INSERT (B,C) puis COMMIT → table = {(B,C)}.

Étape 3 — UPDATE Et2='Z' WHERE Et1='B' → (B,C) devient (B,Z). INSERT (B,B) puis COMMIT → table = {(B,Z), (B,B)} (point de validation).

Étape 4 — Insertions (C,A) (C,B) (C,C) puis DELETE FROM ETIQUETTES puis ROLLBACK → retour au dernier COMMIT : table = {(B,Z), (B,B)}.

Étape 5 — Mises à jour finales :

UPDATE Et1='Z' → {(Z,Z), (Z,B)}
UPDATE Et1='B' WHERE Et2='B' → {(Z,Z), (B,B)}
UPDATE Et1='C' WHERE Et2='C' → aucune ligne avec Et2='C' → inchangé
UPDATE Et2='B' WHERE Et1='A' → aucune ligne avec Et1='A' → inchangé
COMMIT

2
Contenu final de la table
ETIQUETTES
Et1 = 'Z'  |  Et2 = 'Z'
Et1 = 'B'  |  Et2 = 'B'

La table contient 2 lignes : (Z, Z) et (B, B).

Exercices de révision PL/SQL

Procédures, fonctions et triggers — énoncés et solutions complètes

Partie 1 — Procédures Ex. 1 à 4

Schéma : Departement(id_dept, nom_dept) — Employe(id_emp, nom, salaire, date_embauche, id_dept) — Historique_Salaire(id_hist, id_emp, ancien_salaire, nouveau_salaire, date_modification)

1
Augmentation de salaire

Reçoit l'identifiant d'un employé et un pourcentage, et augmente son salaire d'autant.

CREATE OR REPLACE PROCEDURE augmenter_salaire (
    p_id_emp IN Employe.id_emp%TYPE,
    p_pourcentage IN NUMBER
)
IS
BEGIN
    UPDATE Employe
    SET salaire = salaire + (salaire * p_pourcentage / 100)
    WHERE id_emp = p_id_emp;

    COMMIT;
END;
/
2
Prime départementale

Augmente de 5 % le salaire de tous les employés d'un département.

CREATE OR REPLACE PROCEDURE prime_departement (
    p_id_dept IN Departement.id_dept%TYPE
)
IS
BEGIN
    UPDATE Employe
    SET salaire = salaire * 1.05
    WHERE id_dept = p_id_dept;

    COMMIT;
END;
/
3
Embauche d'un employé

Ajoute un employé après avoir vérifié que le département existe ; sinon, lève une exception personnalisée.

CREATE OR REPLACE PROCEDURE embaucher_employe (
    p_id_emp IN NUMBER,
    p_nom IN VARCHAR2,
    p_salaire IN NUMBER,
    p_id_dept IN NUMBER
)
IS
    v_count NUMBER;
    dept_inexistant EXCEPTION;
BEGIN
    SELECT COUNT(*)
    INTO v_count
    FROM Departement
    WHERE id_dept = p_id_dept;

    IF v_count = 0 THEN
        RAISE dept_inexistant;
    END IF;

    INSERT INTO Employe(id_emp, nom, salaire, date_embauche, id_dept)
    VALUES(p_id_emp, p_nom, p_salaire, SYSDATE, p_id_dept);

    COMMIT;

EXCEPTION
    WHEN dept_inexistant THEN
        RAISE_APPLICATION_ERROR(-20001, 'Departement inexistant');
END;
/
4
Rapport départemental

Affiche le nombre d'employés, le salaire moyen et le salaire maximum d'un département avec DBMS_OUTPUT.PUT_LINE.

CREATE OR REPLACE PROCEDURE rapport_departement (
    p_id_dept IN NUMBER
)
IS
    v_nb NUMBER;
    v_moyenne NUMBER;
    v_max NUMBER;
BEGIN
    SELECT COUNT(*), AVG(salaire), MAX(salaire)
    INTO v_nb, v_moyenne, v_max
    FROM Employe
    WHERE id_dept = p_id_dept;

    DBMS_OUTPUT.PUT_LINE('Nombre employes : ' || v_nb);
    DBMS_OUTPUT.PUT_LINE('Salaire moyen   : ' || NVL(v_moyenne, 0));
    DBMS_OUTPUT.PUT_LINE('Salaire maximum : ' || NVL(v_max, 0));
END;
/

Partie 2 — Fonctions Ex. 5 à 8

5
Salaire annuel

Retourne le salaire annuel d'un employé (salaire mensuel × 12).

CREATE OR REPLACE FUNCTION salaire_annuel (
    p_id_emp IN NUMBER
)
RETURN NUMBER
IS
    v_salaire Employe.salaire%TYPE;
BEGIN
    SELECT salaire
    INTO v_salaire
    FROM Employe
    WHERE id_emp = p_id_emp;

    RETURN v_salaire * 12;
END;
/
6
Ancienneté

Retourne le nombre d'années d'ancienneté d'un employé.

CREATE OR REPLACE FUNCTION anciennete (
    p_id_emp IN NUMBER
)
RETURN NUMBER
IS
    v_date DATE;
BEGIN
    SELECT date_embauche
    INTO v_date
    FROM Employe
    WHERE id_emp = p_id_emp;

    RETURN TRUNC(MONTHS_BETWEEN(SYSDATE, v_date) / 12);
END;
/
7
Classification salariale

< 1000 → Junior — 1000 à 3000 → Confirmé — > 3000 → Senior.

CREATE OR REPLACE FUNCTION classification_salariale (
    p_salaire IN NUMBER
)
RETURN VARCHAR2
IS
BEGIN
    IF p_salaire < 1000 THEN
        RETURN 'Junior';
    ELSIF p_salaire <= 3000 THEN
        RETURN 'Confirme';
    ELSE
        RETURN 'Senior';
    END IF;
END;
/
8
Masse salariale

Retourne la masse salariale totale d'un département.

CREATE OR REPLACE FUNCTION masse_salariale (
    p_id_dept IN NUMBER
)
RETURN NUMBER
IS
    v_total NUMBER;
BEGIN
    SELECT NVL(SUM(salaire), 0)
    INTO v_total
    FROM Employe
    WHERE id_dept = p_id_dept;

    RETURN v_total;
END;
/

Partie 3 — Triggers Ex. 9 à 12

9
Historisation des salaires

Avant toute modification de salaire, enregistre le changement dans Historique_Salaire. Comme id_hist est une clé primaire, on crée d'abord une séquence.

CREATE SEQUENCE seq_hist
START WITH 1
INCREMENT BY 1;

CREATE OR REPLACE TRIGGER trg_historique_salaire
BEFORE UPDATE OF salaire
ON Employe
FOR EACH ROW
BEGIN
    INSERT INTO Historique_Salaire(
        id_hist, id_emp, ancien_salaire,
        nouveau_salaire, date_modification
    )
    VALUES(
        seq_hist.NEXTVAL, :OLD.id_emp, :OLD.salaire,
        :NEW.salaire, SYSDATE
    );
END;
/
10
Contrôle du salaire

Empêche l'insertion ou la mise à jour d'un salaire négatif.

CREATE OR REPLACE TRIGGER trg_controle_salaire
BEFORE INSERT OR UPDATE OF salaire
ON Employe
FOR EACH ROW
BEGIN
    IF :NEW.salaire < 0 THEN
        RAISE_APPLICATION_ERROR(
            -20002, 'Salaire negatif interdit'
        );
    END IF;
END;
/
11
Date d'embauche automatique

Renseigne date_embauche avec SYSDATE si elle n'est pas fournie à l'insertion.

CREATE OR REPLACE TRIGGER trg_date_embauche
BEFORE INSERT
ON Employe
FOR EACH ROW
BEGIN
    IF :NEW.date_embauche IS NULL THEN
        :NEW.date_embauche := SYSDATE;
    END IF;
END;
/
12
Limitation des augmentations (max 30 %) 
CREATE OR REPLACE TRIGGER trg_limite_augmentation
BEFORE UPDATE OF salaire
ON Employe
FOR EACH ROW
BEGIN
    IF :NEW.salaire > :OLD.salaire * 1.30 THEN
        RAISE_APPLICATION_ERROR(
            -20003,
            'Augmentation superieure a 30% interdite'
        );
    END IF;
END;
/
12b
Suppression protégée (département < 3 employés) 
CREATE OR REPLACE TRIGGER trg_interdire_suppression
BEFORE DELETE
ON Employe
FOR EACH ROW
DECLARE
    v_nb NUMBER;
BEGIN
    SELECT COUNT(*)
    INTO v_nb
    FROM Employe
    WHERE id_dept = :OLD.id_dept;

    IF v_nb < 3 THEN
        RAISE_APPLICATION_ERROR(
            -20004,
            'Suppression interdite : moins de 3 employes dans le departement'
        );
    END IF;
END;
/

Remarque importante : ce trigger peut provoquer une erreur de table mutante (mutating table) car il interroge la table Employe pendant une suppression sur cette même table. Solution classique : utiliser un trigger composé (compound trigger) ou un trigger de niveau instruction (BEFORE DELETE sans FOR EACH ROW).
